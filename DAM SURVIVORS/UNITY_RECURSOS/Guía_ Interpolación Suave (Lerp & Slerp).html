Guía: Interpolación Suave (Lerp & Slerp)

La técnica profesional para eliminar el movimiento y la rotación "robóticos".

Movimiento Suave: `Vector3.Lerp`

Lerp significa Interpolación Lineal. En lugar de teletransportar un objeto a un nuevo punto, `Lerp` lo mueve *hacia* ese punto un pequeño porcentaje cada frame, creando un efecto de aceleración y frenado suave.

// Ideal para suavizar el movimiento de una cámara
public Transform objetivo;
public float velocidadSuavizado = 5f;
private Vector3 offset;

void Start() {
    offset = transform.position - objetivo.position;
}

void LateUpdate() {
    Vector3 posDeseada = objetivo.position + offset;
    // La magia está aquí:
    transform.position = Vector3.Lerp(
        transform.position, // A: Dónde estoy ahora
        posDeseada,         // B: Dónde quiero estar
        velocidadSuavizado * Time.deltaTime // t: Qué % del camino recorro este frame
    );
}

Rotación Suave: `Quaternion.Slerp`

Slerp significa Interpolación Esférica. Es la versión de `Lerp` pero para rotaciones. Gira un objeto suavemente desde su rotación actual hasta una rotación objetivo.

// Nuestra rotación de jugador usará esto
public float velocidadRotacion = 10f;
Vector3 direccionMovimiento; // (calculada desde el Input System)

void Update() {
    // ... código de movimiento ...

    if (direccionMovimiento != Vector3.zero)
    {
        // 1. Calcular la rotación objetivo
        Quaternion rotacionObjetivo = Quaternion.LookRotation(direccionMovimiento);
        
        // 2. Aplicar la rotación suave
        transform.rotation = Quaternion.Slerp(
            transform.rotation,    // A: Dónde estoy mirando ahora
            rotacionObjetivo,      // B: Dónde quiero mirar
            velocidadRotacion * Time.deltaTime // t: Qué % del giro hago este frame
        );
    }
}

El Creador de Rotación: `Quaternion.LookRotation`

Esta es la función que usamos para crear la "rotación objetivo" (`B`) en el ejemplo anterior. Es increíblemente simple y potente: le das un vector de dirección (un `Vector3`) y te devuelve la rotación (`Quaternion`) que mira en esa dirección.

// Dirección (calculada desde el input, o restando posiciones)
Vector3 direccion = new Vector3(0, 0, 1); // Mirar hacia "adelante" (Z positivo)

// Crear la rotación
Quaternion rotacionHaciaAdelante = Quaternion.LookRotation(direccion);

¡PELIGRO! El Vector Cero: `Quaternion.LookRotation` no sabe cómo "mirar" a la dirección `(0, 0, 0)`. Si le pasas este vector (lo que ocurre si el jugador no está pulsando ninguna tecla), el juego lanzará un error. Por eso en nuestro script de jugador **SIEMPRE** lo pondremos dentro de un `if (direccionMovimiento != Vector3.zero)`.
